ntf_modular/code/controllers/subsystem/ssdaylight.dm

/proc/get_daylight_overlay_appearanceShared overlay that mirrors the single daylight source (via render_source) onto the lighting plane, additively. Because it's a render_source mirror, animating that one source updates the daylight on every turf at once. strength (0-255) scales it: 255 on full daylight turfs, less on the leak rings that feather into indoors. Cached per strength so add_overlay/cut_overlay always match.
/obj/daylight_wash_sourceThe shared daylight source. A single off-screen tile whose colour+alpha track the daylight cycle (animated in ONE place, see update_wash) and whose appearance is published as DAYLIGHT_WASH_RENDER_TARGET. Every daylight turf carries an overlay that render_source-mirrors this, so one animation drives the light on every turf. This is the engine's starlight pattern - see /obj/starlight_appearance.
/atom/movable/screen/plane_master/daylight_anchorDraws nothing. Its only job is to register the shared daylight source onto each viewer's screen (via the HUD reuse system), so the source renders and publishes DAYLIGHT_WASH_RENDER_TARGET for the per-turf overlays to mirror.
